# Yahatl.Domain

This project contains the core domain entities, value objects, and interfaces for YAHATL.

## Purpose

The Domain layer represents the business logic and domain model of the application. It should have no dependencies on external frameworks or infrastructure concerns.

## Contents (to be implemented)

- **Entities**: Note, User, Household, Tag, Trigger, Blocker, Behaviour
- **Value Objects**: Domain-specific value types
- **Interfaces**: Repository interfaces, service contracts
- **Domain Events**: Events that occur within the domain

## Principles

- No dependencies on other projects (except standard .NET libraries)
- Pure C# code representing business concepts
- Framework-agnostic
